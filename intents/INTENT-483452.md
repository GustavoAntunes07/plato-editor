# Description

Add a terminal `closed` state for intents, a CLI command to close an intent intentionally, and shorter IDs for newly created intents.

# Acceptance Criteria

- [x] Lifecycle config supports `closed` as a terminal intent status
- [x] CLI validation accepts `closed` intents
- [x] CLI exposes a command to close an intent with an optional reason
- [x] Newly created intent IDs are shorter than the current 6-digit numeric IDs
- [x] Branch names generated for new intents use the shorter ID
- [x] Tests cover closed intents, close command behavior, and shorter ID generation

# Implementation Notes

- Added `closed` to the `IntentStatus` union, valid statuses, default transitions, terminal statuses, and generated default config.
- Updated `.aidlc/config.yaml` so active intents can transition to `closed`, and `closed` is terminal.
- Added `bun scripts/aidlc.ts close <intent-id|file> [reason]` to close non-terminal intents without requiring Definition of Done.
- Added optional `closed_reason` frontmatter insertion when a close reason is provided.
- Changed `new` intent generation to use a short base36 ID instead of the previous 6-digit timestamp suffix.
- Updated generated branch names to use the new short ID.
- Added `aidlc:close` to `package.json`.
- Updated `AIDLC.md` and `AGENTS.md` to document `closed`, short IDs, and the close command.
